What this is

Code is the system of record.
Figma is where we discuss it.

I'm rebuilding TDS as an AI-native, Mantine-free component library — every component on the same foundation, every component shippable with documentation an AI agent can implement from.

Foundation

React Aria → ids-core → TDS

Every component shares the same a11y + theming primitive stack. No more Mantine underneath.

Documentation

Per-component core.md contract

Intent, anatomy, tokens, states, a11y, do/don't — in a form an AI agent can implement from.

Operating model

MCP pulls from code → renders Figma

Code drives design. Figma becomes the discussion layer, not the source of truth.

Where we're starting from

The system we have today

Today
  • Mantine under everything — third-party dependency under the whole component API
  • Design and code drift — Figma and implementation maintained separately; they disagree
  • No machine-readable contract — nothing describes a component's intent in a form an AI can use
  • Slow net-new work — each component re-litigates tokens, a11y, locale, and motion from scratch
The opportunity
  • ids-core exists — a maintained a11y primitive layer wrapping and re-exporting React Aria
  • AI tooling is ready — agents can implement from well-written contracts today
  • Mantine-free path is clear — we've already rebuilt MenuBase, Stepper, Drawer, Multiselect
  • MCP layer — can pull from code to build and mock pages in Figma

Lock-in

Mantine sits under every component; the system isn't ours end-to-end.

Drift

Figma and code drift apart; neither is canonical. Both are maintained separately.

Scale blocker

Each net-new component re-litigates tokens, theming, locale, a11y, motion, and loading states from scratch. Designers have no sanctioned way to extend.

The solution

An AI-native component library,
full feature set baked in

Tokens + theming

Design tokens, density, dark mode

Compact / regular density tokens, shared easing + duration scale, reduced-motion out of the box.

A11y + locale

Accessible, keyboard navigable, i18n-ready

React Aria handles ARIA patterns. Locale and i18n wired into every component from the start.

core.md

AI-readable component contract

Per-component doc: intent, anatomy, tokens, states, a11y, do/don't. An agent can implement from it.
